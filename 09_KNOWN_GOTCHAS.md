# 09 — Known Gotchas

Hard-won lessons from the original Inspection Intel build. Every entry here represents a real bug that wasted hours. Read this before you start, not after.

## Pipeline gotchas

### yfinance + macOS DNS

`yf.download(..., threads=True)` parallelizes DNS lookups. macOS' resolver chokes on parallel requests and the pipeline silently hangs. **Always pass `threads=False`.** The shipped scripts already do this — don't "optimize" by re-enabling threads.

### Stale `__pycache__`

After editing any Python script the LaunchAgent runs, clear `__pycache__/`. Otherwise the agent runs old bytecode and you see "but I just fixed that" behavior.

```bash
find . -name __pycache__ -type d -exec rm -rf {} +
```

### Block-buffered stdout when piped

Python's stdout is block-buffered when piped to a file. A long pipeline step looks hung when it's actually working. Use `python3 -u` or set `PYTHONUNBUFFERED=1` in the LaunchAgent env.

### Transcript summarizer subprocess timeout

`refresh_inspection_dashboard.py` has an embedded `subprocess.run(timeout=600)` for the transcript summarizer. Bulk loads (>10 transcripts) exceed this and **silently fail** — the subprocess is killed, but the parent doesn't notice.

For bulk transcript loads, run `summarize_transcripts.py` directly:

```bash
python3 _scripts/summarize_transcripts.py --recent 500
```

### CRLF in .sh scripts

If you edit a `.sh` script on Windows or paste into an editor that uses CRLF endings, paths constructed from variables spawn phantom `<CR>` directories. Check with `file morning_pipeline.sh` and fix:

```bash
tr -d '\r' < morning_pipeline.sh > morning_pipeline.sh.tmp && mv morning_pipeline.sh.tmp morning_pipeline.sh
```

### Conflict markers in JSON

After `git merge` or `git rebase`, scan all JSON data files for conflict markers (`<<<<<<<`, `=======`, `>>>>>>>`). They will silently break the dashboard renderer — Python won't error, it'll just emit broken HTML.

```bash
grep -rn '<<<<<<<\|=======\|>>>>>>>' Dashboard/*.json Industry_Data/*.json
```

### Pipeline failure trap

Every shell script that orchestrates the pipeline must end with:

```bash
trap notify_failure EXIT
```

Without it, silent fails reach production and nobody notices for days. Success-path-only emails are not enough.

## Data accuracy gotchas

### Q4 financials are always derived

Q4 numbers are not reported directly. They are `FY − Q1 − Q2 − Q3`. Always enter the full-year value from the 10-K and use an Excel formula for Q4 — never type Q4 directly.

### Adjusted EBITDA alongside reported EBITDA

Always include Adjusted EBITDA next to reported EBITDA in any financial display or extraction. They diverge meaningfully for many TIC/NDT companies (TISI's Q3 2025 adjusted EBITDA was 28.6% above reported).

### `financials_history.py` argument shape

The shared helper expects either:
- a dict keyed by ticker, or
- a list of ticker strings

**Not** a list of dicts. Passing the wrong shape produces a confusing error inside the helper.

## Dashboard gotchas

### Templates vs live HTML

Every dashboard HTML is regenerated by `refresh_inspection_dashboard.py`. If you edit a `*_Dashboard.html` directly, the next pipeline run silently overwrites it. **Always edit the template** (string literal in the Python script for most pages, or `earnings_template.html` for the earnings page).

### Page-header inside container

`<div class="page-header">` must be **inside** `<div class="container">`. Outside, it spans full viewport width and looks broken.

`page-header` CSS must include `border-radius: var(--radius)` and `margin-bottom: var(--gap)`.

### Info icon positioning

Tooltips for info icons must use `position: fixed` with JS positioning relative to the icon. **Never** use `position: absolute` inside an overflow container — the tooltip gets clipped.

Use `_shared/fix_info_tooltips.py` (idempotent) to patch existing pages.

### MutationObserver self-trigger loops

A `MutationObserver` that writes to the DOM it's observing creates a self-trigger loop and freezes the browser. Always disconnect during writes:

```javascript
observer.disconnect();
// do DOM writes here
observer.observe(target, { childList: true, subtree: true });
```

### auth.js + document.visibility

Never set `document.body.style.visibility = 'hidden'` during auth validation. On large dashboards (Company Summary has 38K+ lines of HTML), this triggers a Chrome renderer freeze. Validate quietly without hiding content.

### Ticker bar live refresh

Every dashboard page with a ticker container must end with a self-contained IIFE that calls `/api/quotes` every 60s. Without it, prices freeze at the morning snapshot.

Use `_shared/fix_tickers.py` (idempotent, marker `__ticker_live_v1`) to install this pattern.

### Number formatting

Numbers ≥ 1,000 must use comma formatting (`1,234` not `1234`). Currency same. Use `Intl.NumberFormat` or a helper, not `.toString()`.

### Ticker + name columns

Two-column tables: Ticker column = ticker only, Company column = name only. Don't inline "Name (TICKER)" in two-column tables. Inline format is reserved for single-column contexts.

## Cloudflare gotchas

### Per-site KV namespace IDs

Every Pages project needs its own KV namespace. If you copy-paste a `wrangler.toml` from another project and reuse the ID, two sites silently share subscribers. Always run `npx wrangler kv:namespace create SUBSCRIBERS` for each new site.

### Pages secrets drift from local .env

Cloudflare Pages secrets are set in the dashboard, not from `.env`. They drift over time as keys rotate. After rotating any third-party key (Resend, etc.), update **both**:

1. Local `.env` (for pipeline scripts)
2. Cloudflare Pages environment variables (for Pages Functions)

Otherwise the local pipeline works but the live `/api/*` endpoints fail.

### User-Agent rules differ per endpoint

- **Cloudflare APIs and Resend:** require an explicit User-Agent header. Set one.
- **FRED (Federal Reserve):** **silently blocks** custom User-Agents. Use the default curl/requests UA — do NOT set `-A` or a `User-Agent` header.

These rules are opposite. Both apply. Don't homogenize.

## Deployment gotchas

### Always deploy in the same task

Any Cloudflare Pages dashboard edit must be deployed before the task ends. Never leave local changes "for later" — you'll forget, the next pipeline run regenerates the file from the template, your local edit dies, and confusing inconsistencies pile up.

### Wrangler PATH on macOS

Wrangler may not be on PATH for LaunchAgents. The plist must explicitly set:

```xml
<key>EnvironmentVariables</key>
<dict>
  <key>PATH</key>
  <string>/usr/local/bin:/usr/bin:/bin:/opt/homebrew/bin</string>
</dict>
```

Add other tool paths as needed.

## Quick checklist before calling something done

- [ ] Edited the template, not the live HTML
- [ ] Cleared `__pycache__/` if Python was changed
- [ ] Ran the pipeline end-to-end locally
- [ ] All 6 dashboards open without console errors
- [ ] Deployed to Cloudflare Pages
- [ ] Verified the change is live (not just local)
- [ ] No JSON files contain conflict markers
- [ ] Email-on-failure trap is still in `morning_pipeline.sh`
- [ ] No yfinance call has `threads=True`
- [ ] LaunchAgent reloaded if `.plist` changed
